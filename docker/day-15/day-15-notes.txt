Day 15 - Docker Basics
======================

What is Docker?

Docker is an open-source containerization platform that allows developers to package applications along with their dependencies into lightweight containers.

Why Docker?

• Eliminates "It works on my machine" problems.
• Lightweight compared to Virtual Machines.
• Fast deployment.
• Easy application portability.
• Better resource utilization.

Virtual Machine vs Docker

Virtual Machine
---------------
• Runs a complete Operating System.
• Heavyweight.
• Slower startup.
• More resource usage.

Docker Container
----------------
• Shares the host OS kernel.
• Lightweight.
• Starts within seconds.
• Less resource usage.

Important Docker Components

1. Docker Engine
- Core Docker service.

2. Docker Image
- Read-only template used to create containers.

3. Docker Container
- Running instance of an image.

4. Docker Registry
- Stores Docker images.
- Example: Docker Hub.

5. Docker Volume
- Stores persistent data.

6. Docker Network
- Allows communication between containers.
