> These are the main commands used during Day 28.
>
> Replace placeholders such as `<AWS_ACCOUNT_ID>` with your own values.
> Never commit passwords, GitHub runner registration tokens, AWS access keys, or other secrets.

## 5. GitHub Self-Hosted Runner

Create runner directory:

```bash
mkdir actions-runner
cd actions-runner
```

The runner installation and registration commands should be copied directly from:

```text
GitHub Repository
→ Settings
→ Actions
→ Runners
→ New self-hosted runner
```

Do not save the temporary GitHub runner registration token in a public repository.

Start runner:

```bash
cd ~/actions-runner
./run.sh
```

Expected:

```text
Connected to GitHub

Listening for Jobs
```

When CI/CD starts:

```text
Running job: deploy
```

Successful result:

```text
Job deploy completed with result: Succeeded
```

## 7. AWS ECR Repository

Create ECR repository if your IAM identity has permission:

```bash
aws ecr create-repository \
  --repository-name docker-image-dashboard \
  --region eu-north-1 \
  --image-scanning-configuration scanOnPush=true
```

List repositories:

```bash
aws ecr describe-repositories \
  --region eu-north-1
```

## 8. Login to AWS ECR

Use:

```bash
aws ecr get-login-password --region eu-north-1 | \
docker login \
  --username AWS \
  --password-stdin \
  <AWS_ACCOUNT_ID>.dkr.ecr.eu-north-1.amazonaws.com
```

Expected:

```text
Login Succeeded
```

## 10. ECR Docker Image Format

The ECR image follows this format:

```text
<AWS_ACCOUNT_ID>.dkr.ecr.eu-north-1.amazonaws.com/docker-image-dashboard:<IMAGE_TAG>
```

Example using placeholders:

```bash
docker tag docker-image-dashboard:local \
  <AWS_ACCOUNT_ID>.dkr.ecr.eu-north-1.amazonaws.com/docker-image-dashboard:<IMAGE_TAG>
```

Push manually if testing:

```bash
docker push \
  <AWS_ACCOUNT_ID>.dkr.ecr.eu-north-1.amazonaws.com/docker-image-dashboard:<IMAGE_TAG>
```

Normally, GitHub Actions performs the build and push automatically.

## 16. Update Kubernetes Docker Image

The CI/CD pipeline performs a command similar to:

```bash
kubectl set image deployment/docker-dashboard \
  dashboard=<AWS_ACCOUNT_ID>.dkr.ecr.eu-north-1.amazonaws.com/docker-image-dashboard:<IMAGE_TAG>
```

Check rollout:

```bash
kubectl rollout status deployment/docker-dashboard
```

## 17. Create ECR Kubernetes Pull Secret

Get temporary ECR password:

```bash
ECR_PASSWORD=$(aws ecr get-login-password --region eu-north-1)
```

Create Kubernetes registry secret:

```bash
kubectl create secret docker-registry ecr-secret \
  --docker-server=<AWS_ACCOUNT_ID>.dkr.ecr.eu-north-1.amazonaws.com \
  --docker-username=AWS \
  --docker-password="$ECR_PASSWORD"
```

Clear the temporary shell variable:

```bash
unset ECR_PASSWORD
```

Verify:

```bash
kubectl get secret ecr-secret
```

Expected type:

```text
kubernetes.io/dockerconfigjson
```

---

## 18. Add ECR Secret to Deployment

The Kubernetes Deployment should contain:

```yaml
spec:
  imagePullSecrets:
    - name: ecr-secret
```

A live Deployment can also be patched with:

```bash
kubectl patch deployment docker-dashboard \
  -p '{"spec":{"template":{"spec":{"imagePullSecrets":[{"name":"ecr-secret"}]}}}}'
```

## 19. Troubleshoot ImagePullBackOff

Check pods:

```bash
kubectl get pods
```

Describe problematic pod:

```bash
kubectl describe pod <POD_NAME>
```

Check recent Kubernetes events:

```bash
kubectl get events \
  --sort-by=.metadata.creationTimestamp
```

Typical problem:

```text
ImagePullBackOff
```

Possible cause:

```text
Private ECR image
+
Missing/expired authentication
```

Check Deployment:

```bash
kubectl describe deployment docker-dashboard
```
