Day 22 - Jenkins Basics
=======================

What is Jenkins:
- Open source CI/CD automation server
- Automatically builds, tests, deploys code
- Runs on its own server (unlike GitHub Actions)

Installation Steps:
- Launched EC2 instance (t3.micro, Ubuntu)
- Installed Java 21 (Jenkins requires Java 21+)
- Added Jenkins repo and installed Jenkins
- Started Jenkins with systemctl
- Opened port 8080 in AWS Security Group

Key Commands:
- sudo systemctl start jenkins
- sudo systemctl enable jenkins
- sudo systemctl status jenkins
- sudo systemctl restart jenkins
- sudo cat /var/lib/jenkins/secrets/initialAdminPassword

Jenkins Setup:
- Accessed Jenkins at http://13.48.130.61:8080
- Unlocked with initial admin password
- Installed suggested plugins
- Created admin user

First Job:
- Created freestyle project "my-first-job"
- Added Execute Shell build step
- Ran echo commands successfully
- Saw green checkmark in build history

Troubleshooting Done:
- Fixed Java version (needed Java 21 not 17)
- Fixed disk space (increased EC2 volume to 20GB)
- Fixed node offline issue via Script Console
- Lowered disk space monitoring thresholds
